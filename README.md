# キャッシュフロー本丸（印籠システム）

口座CSVを入口に、案件（売買・金消・個人会社貸借・税）単位で契約・入出金を紐づけ、印刷可能な「印籠」証拠と未来CF予測を出します。オフィスマップはモチベーション用で、CFの真実性には関与しません。

**注意**: 社内の可視化・牽制用です。法令上のAML届出や銀行API連携そのものではありません。

## 最短フロー（Phase 1）

1. **口座・CSV** — 口座マスタを追加 → ネットバンクのCSVを選択 → 列マッピング確認 → 取込確定  
   - 取込行は **削除不可**（イミュータブル）。重複は行ハッシュでスキップ  
   - 訂正は相殺行の再取込で行う想定  
2. **案件ボード** — 売買 / 金消 / 個人会社貸借 / 税の案件を作成（契約日・額・相手・任意の予定）  
3. **明細突合** — 未突合の銀行明細を選び、案件に手動紐づけ  
4. **案件フロー** — 契約・予定・実績の三点を時系列で確認（差額・未突合は赤）  
5. **印籠レポート** — 印刷（ブラウザのPDF）または突合CSV。生成日時＋内容ハッシュ付き

## CSV手順

1. 銀行サイトから明細CSVをダウンロード（UTF-8推奨。BOM付きも可）  
2. 「取込先口座」を選ぶ  
3. ファイル選択後、日付／摘要／金額（または入金・出金分離）／残高列を確認・修正  
4. 必要なら「50万円以上を要説明フラグ」をオンのまま取込確定  

汎用マッピングです。三菱UFJ・みずほ・楽天など列名が違っても、手動で列を合わせられます。

## データ（localStorage）

| キー／フィールド | 内容 |
|---|---|
| `accounts` | 口座マスタ |
| `bankTx` | CSV由来明細（hash・batchId・immutable） |
| `cases` | 案件（sale / loan / related_loan / tax） |
| `schedules` | 予定入出金・利息（予測ソース） |
| `caseLinks` | 明細↔案件の突合 |
| `relatedParties` | 個人会社・関連者 |
| `importBatches` | 取込履歴 |
| `auditLog` | 操作ログ |
| `tx`（従来） | 手入力台帳・精算クエスト用（維持） |

- 金消の元利スケジュール案（案件保存時に登録）
